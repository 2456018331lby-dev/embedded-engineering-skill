## 项目简述

设计一个无线传感器节点，采集温湿度数据并通过 2.4 GHz 无线协议（Wi-Fi / 自定义 FSK）上传至网关。硬件平台为 ESP32，PCB 为双层 FR4（εr = 4.4，厚度 1.6 mm，1 oz 铜），天线形式为板载贴片天线，通过 U.FL 连接器预留外置天线接口。原型阶段，批量 < 10 pcs。

---

## RF 设计流程

整个流程分五步，严格按顺序执行。每步的输出 JSON 作为下一步的输入。

```
calc_antenna → calc_microstrip → calc_cpwg → calc_matching → check_rf_rules
```

## Step 1：天线初始尺寸估算

**目标** 确定板载贴片天线的物理尺寸和馈电阻抗，作为馈线和匹配网络设计的输入。

**命令**
```bash
python3 calc_antenna.py \
  --type patch \
  --freq-ghz 2.4 \
  --er 4.4 \
  --h-mm 1.6 \
  --substrate FR4
```

**计算结果**

| 参数 | 数值 |
|------|------|
| 贴片宽度 W | 38.01 mm |
| 贴片长度 L | 29.42 mm |
| 边缘馈电阻抗 | 307 Ω |
| 50Ω 内嵌深度 | 10.82 mm |
| 典型增益 | 6.5 dBi |
| 典型带宽（VSWR≤2） | ~3.5 % |

**关键决策点**

边缘馈电阻抗为 307 Ω，远高于 50 Ω 系统阻抗。有两种处理方案：

方案 A（推荐用于原型）：使用内嵌馈电（inset feed），将馈点向贴片内部移入 10.82 mm，直接在版图上实现 50 Ω 匹配，省略独立匹配网络。

方案 B：保持边缘馈电，在馈线与贴片之间插入 L 型匹配网络（见 Step 4）。本工程选用方案 B 演示完整流程。

**天线净空要求**：λ₀/10 = 12.5 mm，即贴片四周 12.5 mm 范围内不得有金属走线、过孔或器件。

## Step 2：50 Ω 微带馈线设计

**目标** 确定从 RF 芯片输出到天线馈点的 50 Ω 微带线线宽。

**命令**
```bash
python3 calc_microstrip.py \
  --er 4.4 \
  --h-mm 1.6 \
  --target-z0 50 \
  --freq-ghz 2.4 \
  --substrate FR4 \
  --loss-tangent 0.02 \
  --copper-oz 1
```

**计算结果**

| 参数 | 数值 |
|------|------|
| 线宽 | **3.08 mm** |
| 实际 Z0 | 50.00 Ω |
| 有效介电常数 εeff | 3.33 |
| 导波波长 λg | 68.43 mm |
| λ/4 长度 | 17.11 mm |
| λ/2 长度 | 34.21 mm |
| 导体损耗 | 0.018 dB/cm |
| 介质损耗 | 0.006 dB/cm |
| 总损耗 | **0.024 dB/cm** |

**设计要点** 3.08 mm 线宽在 FR4 1.6 mm 板上是标准值，主流 PCB 厂商均可实现。总损耗 0.024 dB/cm 属于极低水平，原型阶段馈线长度控制在 50 mm 以内时总插损 < 0.12 dB，可接受。

## Step 3：U.FL 连接器 CPWG 馈线设计

**目标** 设计从 RF 芯片到 U.FL 连接器的 CPWG 走线，并确定 via fence 间距。

CPWG 比微带线更适合 U.FL / SMA 边缘连接器过渡，因为其上下左右都有地参考，过渡不连续性更小。

**命令（第一次尝试，参数错误）**
```bash
python3 calc_cpwg.py \
  --er 4.4 --h-mm 1.6 \
  --width-mm 1.8 --gap-mm 0.3 \
  --freq-ghz 2.4 --substrate FR4
# → Z0 = 90.25 Ω  ← FAIL：偏离 50 Ω 太多
```

**几何参数迭代**（手动扫描 width / gap 组合）

| 线宽 w | 间隙 gap | Z0 |
|--------|----------|----|
| 3.5 mm | 0.3 mm | 59.7 Ω |
| 3.5 mm | 0.5 mm | 56.4 Ω |
| **4.0 mm** | **0.5 mm** | **50.3 Ω ✓** |
| 4.5 mm | 0.3 mm | 47.5 Ω |

**最终命令（修正后）**
```bash
python3 calc_cpwg.py \
  --er 4.4 --h-mm 1.6 \
  --width-mm 4.0 --gap-mm 0.5 \
  --freq-ghz 2.4 --substrate FR4
```

**计算结果**

| 参数 | 数值 |
|------|------|
| 线宽 / 间隙 | 4.0 mm / 0.5 mm |
| Z0 | **50.30 Ω** |
| 有效介电常数 εeff | 2.70 |
| λg | 76.02 mm |
| λ/4 长度 | 19.01 mm |
| Via fence 最大间距 | **≤ 3.80 mm**（λg/20）|
| 板边最小净空 | 12.0 mm |

**布局要求** CPWG 两侧各摆一排 via fence，间距 ≤ 3.80 mm（实际布局中用 3.5 mm 满足该要求并留有余量）。CPWG 走线离板边至少 12 mm，U.FL 座子落在 CPWG 末端，地焊盘直接连通 via fence。

## Step 4：匹配网络设计

**目标** 将贴片天线边缘馈电阻抗（307 Ω）匹配至 50 Ω 馈线。

> 若版图采用内嵌馈电（Step 1 方案 A），可跳过此步。

**命令**
```bash
python3 calc_matching.py \
  --type l_network \
  --rs 307 \
  --rl 50 \
  --freq-ghz 2.4
```

**计算结果（低通拓扑：并联 C → 串联 L）**

| 元件 | 位置 | 类型 | 值 |
|------|------|------|----|
| C1 | 并联（天线侧） | 电容 | **0.49 pF** |
| L1 | 串联（馈线侧） | 电感 | **7.52 nH** |

网络 Q = 2.27，满足 MN-01 规则（Q ≤ 10）。两个元件值均在 0402 器件可采购范围内。

**备选高通拓扑（并联 L → 串联 C）**：适合需要 DC 隔离或谐波抑制要求较高的场景。

**器件选型参考**：C1 ≈ 0.49 pF，推荐选取 0.5 pF（Murata GJM0335C1ER50 或等效）。L1 ≈ 7.52 nH，推荐选取 7.5 nH（Murata LQG15HS7N5 或等效）。实装后用 VNA 调优。

## 验证方法

PCB 加工回来后，按以下步骤逐项验证，确认设计是否达标。

### 第一轮验证：上电前

目视检查贴片天线四周净空是否存在意外走线或过孔侵入。确认 via fence 布局是否按设计连续分布在 CPWG 两侧。确认 U.FL 焊盘地脚是否与 via fence 直接连通，无长走线。确认去耦电容位置是否紧邻 RF IC 供电引脚。

### 第二轮验证：用 VNA 测 S11（最关键）

将 VNA PORT 1 连接到 U.FL 接口，扫频范围设为 2.0–3.0 GHz，步进 1 MHz 以上。

判断标准如下：

S11 在 2.4 GHz 处 ≤ −10 dB（即反射 < 10 %），表示天线匹配可用。S11 在 2.4–2.48 GHz 全段均 ≤ −10 dB，表示带宽覆盖主流 2.4 GHz 协议。若 S11 谐振点偏移（频率偏高：贴片偏小或 εr 偏低；频率偏低：贴片偏大或 εr 偏高），按以下方式调整：

谐振频率偏高 → 沿贴片长度方向在开路端焊接一小段铜箔或加载网络，或者物理打磨缩短电气路径（不推荐）。谐振频率偏低 → 用铣刀或激光修短贴片长度，每次修 0.5 mm，重测 S11，直到谐振在 2.44 GHz 附近。匹配不佳（S11 谐振很深但不在 50 Ω 系统中）→ 调整 C1 / L1 值，可在焊盘上堆焊不同值的 0402 器件逐步逼近。

### 第三轮验证：功能性射频测试

将节点上电，用频谱仪在 2.44 GHz 观测发射功率，确认发射频率和功率等级是否符合协议规范。若使用 Wi-Fi，用 iPerf3 或路由器 RSSI 读数确认链路质量。若使用自定义协议，用另一块相同节点或软件无线电（SDR）接收，在 3 米距离下确认 PER（误包率）< 1 %。

## 常见问题排查

S11 谐振存在但 S11_min 仅 −6 dB 左右：通常是匹配网络元件值偏差，微调 C1（±0.1 pF）或 L1（±0.5 nH）。

S11 无明显谐振（曲线平坦）：检查天线馈线是否断路或短路，用万用表测量 U.FL 到贴片馈点的直流通断。

发射频率偏移：ESP32 的晶振误差或温度漂移，检查晶振规格，确认是否需要 TCXO。

近场干扰（RSSI 在节点附近极差）：检查天线净空区是否被金属机壳或电池遮挡，调整天线位置或方向。
